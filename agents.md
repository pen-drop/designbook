# Project Rules

## CLI — Development Override

Skills and workflows always use `npx storybook-addon-designbook` as the canonical CLI form.
**During development** the addon is not published, so you MUST replace every occurrence with the local build:

| In skills / workflows | Execute as |
|---|---|
| `npx storybook-addon-designbook <cmd>` | `node packages/storybook-addon-designbook/dist/cli.js <cmd>` |

This substitution applies to ALL commands: `workflow create`, `workflow update`, `workflow validate`, `validate data-model`, `validate tokens`, `validate component`, `validate data`, `config`, etc.

## Generated Files — Not in Git

The test integration outputs are gitignored because they are generated by Designbook workflows (`/debo-*`):

- `packages/integrations/test-integration-drupal/designbook/` — scenes, data, entity-mapping, tokens, data-model
- `packages/integrations/test-integration-drupal/components/` — SDC components (heading, card, badge, etc.)

These directories are populated by running workflows and are not version-controlled. Do not try to `git add` them.

## Path Configuration

All paths inside `designbook` skills and `debo` workflows are relative to `packages/integrations/test-integration-drupal/`.

## Skill References

### Workflows — `designbook-workflow`

When executing any `debo-*` workflow, ALWAYS load and follow the `designbook-workflow` skill before processing any steps:
- Read the `workflow:` frontmatter block and run `workflow create` to initialize tracking
- Process every `!WORKFLOW_FILE` marker with status updates and validation
- Process `!WORKFLOW_DONE` to archive the workflow on completion

**REQUIRED: If the workflow frontmatter contains a `workflow:` block with `tasks:`, you MUST run `workflow create` before doing any work.** No exceptions — even for short or conversational workflows.

### Skills erstellen/ändern — `skill-creator`

Beim Anlegen neuer Skills oder beim Ändern bestehender Skills MUSS der `skill-creator` Skill verwendet werden. Keine Ausnahmen.

### Komponenten — framework-spezifische Skills

When planning, designing, or writing anything involving Designbook components:
1. Read `designbook.config.yml` to determine `DESIGNBOOK_FRAMEWORK_COMPONENT` and `DESIGNBOOK_FRAMEWORK_CSS`
2. Load ALL matching skills:
   - `designbook-components-[DESIGNBOOK_FRAMEWORK_COMPONENT]` (e.g. `designbook-components-sdc`)
   - `designbook-css-[DESIGNBOOK_FRAMEWORK_CSS]` (e.g. `designbook-css-daisyui`)
   - `designbook-css-generate`
3. Apply skill rules (naming, layout, variants) in all design documents and proposals

```bash
cd packages/storybook-addon-designbook && npx eslint --cache --fix . && cd ../..
npm run lint
```
