# Designbook

## Description
Designbook is an AI-powered design-to-implementation platform that bridges the gap between design tools and CMS/frontend development. It transforms designs from any source into structured, implementation-ready specifications through guided workflows.

## Problems & Solutions

### Problem 1: Manual Design-to-Code Translation
Translating designs into CMS content types and components is tedious manual work. Designbook solves this with guided AI workflows that progressively structure designs into implementation-ready specs — from vision through data model to fully rendered screens.

### Problem 2: Lost Context in Design Handoff
Design handoff loses context between designers and developers. Designbook provides a single source of truth with YAML specs, component code, and live Storybook preview, keeping design intent intact throughout implementation.

### Problem 3: Framework-Specific Implementation Patterns
Every CMS and frontend framework requires different implementation patterns. Designbook takes a framework-agnostic approach that generates output for any stack — Drupal, React, Twig, Tailwind, and more — through swappable integration plugins.

## Key Features
- Staged AI workflows (vision, sections, tokens, data model, sample data, shell, screens)
- Design input from Figma, Google Stitch, existing websites, or AI conversation
- Live component preview in Storybook
- YAML-based specs for content types, view modes, and page compositions
- Integration plugins for Drupal, Tailwind CSS, and SDC
