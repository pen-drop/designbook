# Development Rules

## CLI — Development Override

Skills and workflows always use `npx storybook-addon-designbook` as the canonical CLI form.
**During development** the addon is not published, so you MUST replace every occurrence with the local build:

| In skills / workflows | Execute as |
|---|---|
| `npx storybook-addon-designbook <cmd>` | `node packages/storybook-addon-designbook/dist/cli.js <cmd>` |

This substitution applies to ALL commands: `workflow create`, `workflow update`, `workflow validate`, `validate data-model`, `validate tokens`, `validate component`, `validate data`, `config`, etc.

## Generated Files — Not in Git

The test integration outputs are gitignored because they are generated by Designbook workflows (`/debo-*`):

- `packages/integrations/test-integration-drupal/designbook/` — scenes, data, view-modes, tokens, data-model
- `packages/integrations/test-integration-drupal/components/` — SDC components (heading, card, badge, etc.)

These directories are populated by running workflows and are not version-controlled. Do not try to `git add` them.

## Linting — MUST Pass Before Committing

```bash
cd packages/storybook-addon-designbook && npx eslint --cache --fix . && cd ../..
npm run lint
```
