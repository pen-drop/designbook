# Execute Transformations

This step runs all `.jsonata` expression files generated by the framework skill to produce CSS output files.

> Requires `jsonata-w@1.0.1+` which supports direct string output.

## Prerequisites
- Step 3: Delegate to Framework Skill (completed)
- `.jsonata` files exist in `$DESIGNBOOK_DIST/designbook-css-[framework]/`

## Process

### 1. Ensure output directories exist

```bash
mkdir -p $DESIGNBOOK_DRUPAL_THEME/css/tokens
mkdir -p $DESIGNBOOK_DRUPAL_THEME/css/themes
```

### 2. Execute all transformations

```bash
for f in $DESIGNBOOK_DIST/designbook-css-$DESIGNBOOK_CSS_FRAMEWORK/generate-*.jsonata; do
  npx jsonata-w transform "$f"
done
```

Each transformation directly produces its `.src.css` file — no intermediate files needed. The `@config` block inside each `.jsonata` file specifies the input and output paths.

### 3. Verify execution

- Check exit code of each transformation
- Report any errors with the specific `.jsonata` file that failed

## Error Handling
- Transformation fails: Show jsonata-w error output and the failing expression file
- No `.jsonata` files found: Report that the framework skill did not generate any expressions
- Permission issues: Show permissions error for output directory

## Technical Notes
- jsonata-w 1.0.1+ writes string results directly as raw text (no JSON wrapping)
- Each `.jsonata` file is self-contained with embedded `@config` for input/output paths
- Use `npx jsonata-w inspect $DESIGNBOOK_DIST/design-tokens.json --summary` to debug token structure
