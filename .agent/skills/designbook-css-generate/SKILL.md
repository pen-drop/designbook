# Designbook CSS Generate

Generic CSS generation pipeline that transforms W3C Design Tokens into CSS files. This skill orchestrates the full pipeline and delegates framework-specific expression generation to the appropriate CSS framework skill.

## Prerequisites

1. `DESIGNBOOK_FRAMEWORK_CSS` must be set (via `frameworks.css` in `designbook.config.yml`)
2. `DESIGNBOOK_DRUPAL_THEME` must be set (via `drupal.theme` in `designbook.config.yml`)
3. `DESIGNBOOK_DIST` must be set
4. Load configuration:
   ```bash
   eval "$(node packages/storybook-addon-designbook/dist/cli.js config)"
   ```

## Capability

### Generate CSS
**Trigger**: When asked to "generate CSS", "create CSS files", "update CSS", "regenerate CSS", or "export CSS from tokens".

**Action**: Execute the following steps in order:

1. **Verify Input** (`./steps/verify-input.md`)
   - Checks that `$DESIGNBOOK_DIST/design-tokens.yml` exists
   - Validates W3C token file is valid JSON and not empty

2. **Check Regeneration** (`./steps/check-regeneration.md`)
   - Determines if regeneration is needed
   - Compares file timestamps (optimization)
   - Allows force regeneration

3. **Delegate to Framework Skill** (`./steps/delegate-framework.md`)
   - Selects the framework-specific skill based on `DESIGNBOOK_FRAMEWORK_CSS`
   - The framework skill generates `.jsonata` expression files
   - Each expression file is self-contained with input/output paths

4. **Execute Transformations** (`./steps/execute-transforms.md`)
   - Ensures output directories exist
   - Runs all generated `.jsonata` files via `jsonata-w transform`
   - Each transformation produces a `.src.css` file directly

5. **Ensure CSS Imports** (`./steps/ensure-imports.md`)
   - Updates `$DESIGNBOOK_DRUPAL_THEME/css/app.src.css` with `@import` lines
   - Adds missing imports for generated token and theme files

6. **Verify Output** (`./steps/verify-output.md`)
   - Confirms all CSS files exist and are non-empty
   - Validates CSS syntax
   - Reports generation statistics

## Parameters
- None required (processes all tokens from W3C file)
- Optional: `force` flag to skip regeneration check

## Context
- **Input**: `$DESIGNBOOK_DIST/design-tokens.yml` (W3C Design Tokens)
- **Expressions**: `$DESIGNBOOK_DIST/designbook-css-[framework]/*.jsonata` (generated by framework skill)
- **Output**: `$DESIGNBOOK_DRUPAL_THEME/css/tokens/*.src.css` and `$DESIGNBOOK_DRUPAL_THEME/css/themes/*.src.css`

## Output Structure
```
$DESIGNBOOK_DRUPAL_THEME/css/
├── tokens/
│   └── *.src.css       (token files, generated by framework skill)
└── themes/
    └── *.src.css       (theme files, generated by framework skill)
```

## Error Handling
Each step handles its own errors and provides clear feedback:
- Missing token file: Prompts to run Designbook Tokens skill first
- No framework configured: Shows configuration instructions
- Unknown framework: Lists supported frameworks
- Script errors: Shows jsonata-w error details
- Invalid output: Shows validation errors

## Optimization

This pipeline includes optimization via timestamp checking:
- Skips regeneration if CSS files are up-to-date
- Saves processing time
- Can be overridden with force flag

## Next Steps
After generating CSS:
- Review generated CSS files in `css/tokens/` and `css/themes/`
- Test CSS tokens in components
- Compile CSS with your build tool (e.g. Tailwind)
- Update design system documentation
